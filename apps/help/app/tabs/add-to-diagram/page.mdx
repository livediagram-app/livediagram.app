Sometimes a tab you built in one diagram belongs in another too: a glossary, a shared timeline, an architecture board two projects both rely on. **Add to Diagram** adds the active tab to another diagram you own, without leaving the editor.

## Adding a tab to another diagram

<Step n={1}>Open the active tab's ellipsis menu and expand the **Organise** section.</Step>
<Step n={2}>Choose **Add to Diagram**.</Step>
<Step n={3}>Pick the destination diagram from your list. The tab is added to it.</Step>

The option is only available when you actually have somewhere to add it to: if this is your only diagram, **Add to Diagram** is greyed out until you create another.

<Figure caption="Add to Diagram in the tab menu opens a picker listing your other diagrams to add the tab to.">

</Figure>

## One tab, shared between diagrams

Add to Diagram doesn't make a copy: it is the same tab in both places. Edit its elements, restyle it, or rename it from either diagram and the change lands in the other as well. If the other diagram is already open in another window, reload it to see the latest edits.

Two things stay per-diagram, so each diagram can organise the shared tab its own way: the tab's position in the tab bar, and which [tab folder](/help/tabs/tab-folders/) (if any) it sits in.

Removing the tab from one diagram doesn't touch the others; the tab and its contents are only deleted for good when you remove it from the last diagram containing it.

<Tip>
  Prefer an independent copy you can tailor per project? Use **Duplicate** in the same menu first,
  then add the duplicate to the other diagram.
</Tip>
